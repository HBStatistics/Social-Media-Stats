Social Media Stats Unified API

Search creators, fetch cross-platform stats, compare channels, and build watchlists. Below is what you can do, how to authenticate, and what is prohibited.

Quick Start

The API base URL is:

https://api.socialmediastats.app/v1

All requests must be authenticated with your API key. Provide it via header:

Authorization: Bearer YOUR_API_KEY

Example search for a creator (YouTube, TikTok, X, etc.):

curl "https://api.socialmediastats.app/v1/search?q=MrBeast" -H "Authorization: Bearer YOUR_API_KEY"

What You Can Do

Designed for data consumers, dashboards, analytics tooling, and internal tracking.
  • Search for creators across YouTube, TikTok, X, Instagram, Bluesky, etc., with fuzzy name or ID matching.
  • Retrieve unified profile summaries (subscribers/followers, views, rank, platform breakdown).
  • Compare multiple creators side-by-side with deltas and growth velocity.
  • Maintain a personal watchlist (requires authentication).
  • Receive projections and recent growth sparklines.
  • Integrate data into dashboards, streams, community tools (with attribution).

What You Cannot Do

  • Share or embed your API key publicly (e.g., in client-side JS without proxying).
  • Excessively poll the API to circumvent rate limits or attempt to farm all data in bulk without permission.
  • Redistribute raw data wholesale without attribution or a license agreement.
  • Reverse-engineer internal ranking/estimation algorithms or pretend to be the official service.
  • Use the API for abusive automation (spam, fake account generation, scraping content platforms at scale).
  • Misrepresent the origin of data (e.g., claim third-party visuals are native if you’re using this API without credit where required).

Authentication

Every request must include your API key. You can supply it via:

  • Header: Authorization: Bearer YOUR_API_KEY
  • Query param (fallback): ?api_key=YOUR_API_KEY (not recommended for public clients)

Unauthorized or missing credentials will return 401 Unauthorized.

Endpoints

GET
/search — fuzzy lookup for creators across platforms
Query multi-platform by name, handle, or ID

Parameters:

ParamTypeDescription
qstringSearch term (name, handle, or partial ID)
platformstringOptional filter: youtube, tiktok, x, instagram, bluesky
limitintMax results (default 7, max 20)

Example:

curl "https://api.socialmediastats.app/v1/search?q=HB%20Statistics&limit=5" -H "Authorization: Bearer YOUR_API_KEY"
GET
/channel/{id} — get unified stats for a single creator
ID can be internal normalized ID or platform-specific (e.g., YouTube channel ID)

Example:

curl "https://api.socialmediastats.app/v1/channel/UCiRU8yhg2-WAcOK_QdlahRw" -H "Authorization: Bearer YOUR_API_KEY"
GET
/compare — compare multiple creators
Supply a comma-separated list of IDs

Parameters:

ParamTypeDescription
idsstringComma-separated creator IDs

Example:

curl "https://api.socialmediastats.app/v1/compare?ids=UCiRU8yhg2-WAcOK_QdlahRw,UCX6OQ3DkcsbYNE6H8uQQuVA" -H "Authorization: Bearer YOUR_API_KEY"
POST
/watchlist — create / update your watchlist (auth required)
Save favorite creators for quick access and alerting

Body:

{
  "watchlist": ["UCiRU8yhg2-WAcOK_QdlahRw", "UCX6OQ3DkcsbYNE6H8uQQuVA"]
}

Rate Limits & Fair Use

To ensure stability for everyone, default limits are:

  • 60 requests per minute per API key.
  • Bulk or high-frequency access may be throttled; use backoff on 429 responses.
  • Caching responses on your side (with appropriate TTL) is encouraged for repeated lookups.

Requests that appear abusive (e.g., repeated identical requests without caching, scraping entire datasets via loops) may be rate-limited or temporarily blocked. Contact support for higher-volume access or custom tiers.

Error Responses

Standard error structure:

{
  "error": {
    "code": 401,
    "message": "Unauthorized: missing or invalid API key",
    "details": null
  }
}
StatusMeaning
400Bad request (invalid parameters)
401Unauthorized (missing/invalid API key)
403Forbidden (access to resource denied)
404Not found (unknown creator or endpoint)
429Rate limit exceeded — back off and retry after Retry-After
500Server error (retry with exponential backoff)
